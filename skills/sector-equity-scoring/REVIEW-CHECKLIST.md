# Formula 审核清单

每个板块 formula 完成后，按以下清单逐项检查：

## 1. 指标完整性
- [ ] 必须 18 指标（18/18 或 17/18），不接受 12 指标
- [ ] 数据来源：财务数据必须用 datacenter-web API (RPT_F10_FINANCE_MAINFINADATA)，不能只用 push2 batch API
- [ ] 权重总和 = 100%（允许 101% 归一化，不接受更大偏差）

## 2. 权重结构
- [ ] 估值 ~20%
- [ ] 盈利能力 ~30%
- [ ] 成长 ~15-25%（周期行业偏低，成长行业偏高）
- [ ] 现金流 ~10-13%
- [ ] 风险 ~10-12%（D/A 不超过 4-5%，三指标均衡分配）
- [ ] 质量 ~6-10%
- [ ] 权重分配逻辑与行业特征匹配

## 3. V2 规则执行
- [ ] 增长率 ±200% 封顶（表格应显示封顶后的值，即 200.0 而非原始值）
- [ ] ROE/ROIC ≤60% 封顶
- [ ] PE<0 排除（亏损股不参与 PE 排名）
- [ ] 覆盖率惩罚：指标 <10 时 score *= count/18
- [ ] ST/*ST 排除
- [ ] B 股排除（9xxxxx 代码）

## 4. 排名合理性
- [ ] 行业龙头排名符合预期（不需要第一，但要在合理区间）
- [ ] 亏损股集中在底部
- [ ] 无 ST 股进入 Top 50
- [ ] 无 B 股重复（A/B 股同一家公司）
- [ ] 特殊高值检查：净利率 >100%、股息率 >10%、ROE >50% 需标注原因

## 5. 分数分布
- [ ] Mean ~49-51（居中合理）
- [ ] Median ~49-52
- [ ] >70 分占比 <15%（区分度）
- [ ] <40 分占比合理（不应过高除非行业整体低迷）

## 6. 数据质量
- [ ] 18/18 覆盖率 >60%
- [ ] 缺失指标集中在合理字段（股息率、CF/NP 对亏损股缺失正常）
- [ ] 无明显数据错误（如资产负债率字段用错 f186 vs f188）

## 7. 子行业覆盖度
- [ ] 子行业分类与申万二级/三级分类一致
- [ ] 各子行业均有代表性股票进入排名
- [ ] 子行业得分差异有合理解释（概述或合理性检验中说明）

## 8. 局限性章节
- [ ] 合理性检验包含"局限性"小节
- [ ] 至少列出 3 条局限性（如：单年增长率 vs CAGR、缺少行业特有指标、纯横截面无纵向对比等）
- [ ] 局限性描述具体、可操作，非泛泛而谈

## 9. 存储前处理
- [ ] 复制到 `formulas/XX-name.md`
- [ ] 运行 `mark_st.py` 标记残留 ST 股 ⛔ST
- [ ] 更新 CHECKLIST.md 进度
